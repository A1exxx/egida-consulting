# Яндекс.Директ — пакет запуска «Эгида»

Бюджет старта: **20 000 ₽/мес**. Гео: **Воронеж + соседние области** (Поиск), РСЯ — шире. Кабинет — тип «Иностранное физлицо» (рекламодатель в Таиланде; НДС 0%; оплата иностранной картой). Фокус старта — кластер **УСН+НДС** (кадры — фаза 2).

## 1. Структура кампаний

```
Аккаунт
├── ЕПК Поиск — «УСН + НДС» (СТАРТ, 12 000 ₽)
│   ├── Группа: усн с ндс / переход на усн ндс
│   ├── Группа: ндс на упрощёнке / как считать
│   └── Группа: ставка ндс 5 7 22 / выбрать

    ├── кадровый аудит / разработка лна / пвтр
    └── юрист трудовые споры работодатель / подготовка к гит
```

## 2. Семантика (стартовое ядро — Поиск, УСН+НДС)

**Горячие коммерческие:**
- усн с ндс консультация / помощь
- переход на усн с ндс
- ндс на упрощёнке как считать
- какую ставку ндс выбрать усн
- ставка ндс 5 или 7 усн
- налоговая реформа 2025 усн ндс
- бухгалтер консультация онлайн усн
- учётная политика усн ндс

**Минус-слова (обязательно):** бесплатно скачать, калькулятор онлайн, форум, вакансия, работа, своими руками, реферат, курсы, обучение, 1с скачать, кодекс, википедия.

**Фаза 2 (кадры):** кадровый аудит, разработка лна, разработка пвтр, трудовой договор составить, юрист трудовые споры работодатель, подготовка к проверке гит, положение о персональных данных сотрудников.

## 4. Стратегия ставок
- **Старт:** оплата за клики («Максимум кликов» с ограничением ставки), пока нет данных.
- После накопления ~10 конверсий/нед → перейти на «Целевую цену конверсии».
- Остаток на счёте для автостратегии: не менее max(5 000 ₽, 3 × цель CPA).

## 5. Цели Яндекс.Метрики (НАСТРОИТЬ ДО ЗАПУСКА)
Поставить счётчик Метрики в `<head>` сайта и создать цели:
1. **Отправка формы** (низ страницы) — событие submit `#leadForm`.
2. **Прохождение квиза** — JS-цель в момент показа экрана успеха.
3. **Клик по телефону** — клик по `a[href^="tel:"]`.
4. **Клик в WhatsApp** — клик по `a[href*="wa.me"]`.

> Без целей автостратегии не обучаются. Счётчик добавить перед запуском кампаний (сейчас на сайте не стоит — открытый пункт).

## 6. Гео
- **Поиск (старт):** Воронеж + Воронежская обл. → при наличии бюджета добавить Липецкую, Белгородскую, Тамбовскую, Курскую.
- **РСЯ:** шире (ЦЧР или вся РФ) — клик дешевле, формат онлайн.
- **Москву не брать** до бюджета 80к+ (клик в 3–4 раза дороже).

## 7. Чек-лист запуска (иностранный кабинет, из Таиланда)
- [ ] Создать аккаунт Яндекс → Директ, тип рекламодателя **«Иностранное физическое лицо»**, код страны Таиланд (ОКСМ 764).
- [ ] Привязать оплату (иностранная Visa/MC, не РФ; первая верификация — скан паспорта, до 3 дней).
- [ ] Поставить счётчик Метрики на сайт + цели (раздел 5).
- [ ] Заполнить «Данные о рекламодателе» (маркировка — Яндекс делает токены сам).
- [ ] Завести кампанию Поиск, гео Воронеж, ядро УСН+НДС, минус-слова, объявления.
- [ ] Пройти модерацию (без слов-гарантий; посадочная = объявлению).
- [ ] Запустить с оплатой за клики, дневной лимит ~600–800 ₽.
- [ ] Через 3–4 недели — анализ, отключить мусорные ключи, перейти на оплату за конверсии.

### Почему «САЙТЫ ЕПК» не заходит (за 30 дней)
| Метрика | Значение | Вывод |
|---|---|---|
| Показы | 22 433 | трафика много |
| Клики | 1 208 | CTR 5.38%, CPC 6.74₽ — очень дёшево = РСЯ |
| Расход | 8 140₽ | |
| **Конверсии** | **3** | **CR 0.25% — катастрофа** |
| Цель | 500₽ за «отправку формы», по факту CPA в гриде 2 713₽ | трекинг конверсий рассинхронён — цифрам нельзя доверять |

**Настройки, которые убивают результат:**
1. **Поиск и РСЯ в одной ЕПК** → дешёвый широкий трафик РСЯ съедает бюджет, горячий поиск тонет.
2. **Стратегия «Максимум конверсий / оплата за конверсии»** при <1 конверсии в неделю. Автостратегии нужно **≥10 конверсий/неделю**, чтобы обучиться. Данных нет → алгоритм льёт в самый дешёвый мусорный трафик.
3. **Гео = вся Россия** + аудитория «вся заинтересованная» + **автотаргетинг включён** → максимально широко, низкий интент.
4. **Конверсия-трекинг мутный** (цель 500₽ vs CPA 2713₽) — невозможно судить о реальных заявках.

Итог: **трафик есть, заявок нет.** Классическая ошибка «запустил ЕПК на автомате на всю РФ».

### Выводы для запуска ЭГИДЫ (исправляем ошибки DEADLINE)
1. **Разделить Поиск и РСЯ** — это разные кампании. Старт = **только Поиск** (горячий интент), РСЯ подключаем потом для ретаргетинга.
2. **НЕ начинать с оплаты за конверсии / Максимум конверсий** — нечем обучать. Старт = **«Максимум кликов» с ограничением ставки** (оплата за клики) на узком ядре. Накопим ≥10 конверсий/нед → тогда переключаемся на оплату за конверсии.
3. **Гео = Воронеж + область**, не вся РФ (дешевле, релевантнее). РСЯ позже — шире.
4. **Автотаргетинг — выключить** на старте (или строго ограничить категориями). Только ручные ключи + минус-слова.
5. **Метрика + цели настроить И ПРОВЕРИТЬ, что срабатывают** (отправка формы, прохождение квиза, клик по тел/WhatsApp). Аккаунт DEADLINE показал: трекинг легко сломать — обязательно валидировать тестовой заявкой.
6. **Копировать кампанию DEADLINE НЕ нужно** — её настройки это анти-паттерн. Строим Эгиду с нуля по структуре выше.

### Подсказка по бенчмаркам из аккаунта
- CPC в РСЯ по их тематике ~6–7₽ (очень дёшево, но мусор). На **Поиске** по консалтингу будет дороже (≈100–200₽ в регионе), но это целевой трафик — лучше 10 горячих кликов, чем 1000 мусорных.
- 7000₽/нед на ЕПК (= ~28к/мес) дали 3 заявки. Наш план 20к/мес на узкий Поиск Воронежа при рабочем трекинге должен дать кратно больше реальных обращений.
